
# Алгоритм преобразования Gerber → G-code для дозатора паяльной пасты

Данный документ описывает логику работы класса `GCodeGenerator`, реализованного в проекте `GerberParser`.  
Цель — автоматическое создание G-code для станка, наносящего паяльную пасту на печатные платы.

## 🧩 Общая схема

Gerber File
↓
[GerberParser] → GerberLayer (polygons, bounds)
↓
GCodeGenerator::generateDispensePoints() → QVector<DispensePoint>
↓
GCodeGenerator::generateGCode() → QString (G-code текст)
↓
G-code файл (.gcode) → запуск на оборудовании


## 1. Входные данные

Класс `GCodeGenerator` принимает:
- `GerberLayer layer` — слой, содержащий:
  - `QVector<QPolygonF> polygons` — список полигонов (апертуры, пады)
  - `QRectF boundingRect` — общие границы изображения

> Предполагается, что Gerber уже распарсен, и полигоны корректно построены.

## 2. Генерация точек дозирования (`generateDispensePoints`)

### 2.1. Вычисление центра полигона (барицентр)

Для каждого полигона вычисляется геометрический центр масс:

```cpp
double area = 0.0;
QPointF center(0, 0);
for (int i = 0; i < poly.size(); ++i) {
    int j = (i + 1) % poly.size();
    double cross = poly[i].x() * poly[j].y() - poly[j].x() * poly[i].y();
    area += cross;
    center += (poly[i] + poly[j]) * cross;
}
area /= 2.0;
center /= (6.0 * area);
📌 Преимущество: подходит для любых выпуклых и вогнутых форм.

2.2. Определение эффективного диаметра
Чтобы избежать переизбытка или недостатка пасты, используется адаптивный подход:

Тип формы	Метод расчёта
Круглая (≥8 вершин, лежат на окружности)	Диаметр = расстояние от центра до вершин × 2
Прямоугольная / овальная	Диаметр = минимальное расстояние от центра до рёбер × 2
Псевдокод:
CPP
rEdge = minDistPointToPolygonEdges(center, poly);  // до сторон
rVert = minDistPointToVertices(center, poly);      // до вершин
effR = isApproxCircular(center, poly) ? rVert : rEdge;
diameter = 2.0 * effR;
✅ Это гарантирует:

Для круглых падов — точное совпадение с апертурой
Для прямоугольных — вписанную окружность (без вытекания)
2.3. Создание точки дозирования
CPP
struct DispensePoint {
    QPointF position;     // X, Y
    double diameter;      // мм
    int dispenseTime;     // мс
    double height;        // Z при дозировании
};
Учитываются:

Смещение по осям (m_offsetX, m_offsetY)
Глубина погружения (m_zDown)
Время дозирования (см. ниже)
2.4. Оптимизация траектории (сортировка)
Точки сортируются "змейкой" по координатам:

CPP
std::sort(points.begin(), points.end(), [](const DispensePoint &a, const DispensePoint &b) {
    if (qAbs(a.position.y() - b.position.y()) > 1.0)
        return a.position.y() < b.position.y();
    return a.position.x() < b.position.x();
});
📌 Эффект: минимизация перемещений → снижение времени обработки.

3. Генерация G-code (generateGCode)
На основе списка DispensePoint генерируется текстовый G-code.

3.1. Заголовок
Содержит метаданные:

GCODE
; ===== G-CODE FOR PASTE DISPENSER =====
; Generated by Gerber_Parser
; Points: 42
; Feed rate: 1500 mm/min
; Z up: 5.000 mm
; Z down: -0.100 mm
; Dwell time: 100 ms
3.2. Инициализация
GCODE
G21 ; Units in mm
G90 ; Absolute positioning
G00 Z5.000 ; Raise Z
G00 X0 Y0 ; Go to origin
M03 S100 ; Start pump
G04 P500 ; Wait for pump
3.3. Цикл по точкам
Для каждой точки генерируется блок:

GCODE
; Point 1 - diameter: 0.8 mm
G00 Z5.000                     ; Подъём Z
G00 X10.200 Y15.300            ; Перемещение к паду
G01 Z-0.100 F750               ; Медленное опускание
M07 ; Valve ON                 ; Открыть клапан
G04 P120                       ; Пауза (дозирование)
M09 ; Valve OFF                ; Закрыть клапан
G00 Z5.000                     ; Подъём
Ключевые команды:
Команда	Назначение
G00	Быстрое перемещение
G01	Линейное движение с подачей
M07	Включить клапан (подаёт пасту)
M09	Выключить клапан
G04 P...	Пауза в миллисекундах
3.4. Завершение программы
GCODE
M05 ; Stop pump
G00 X0 Y0 ; Return to origin
G00 Z5.000
M02 ; End of program
4. Расчёт времени дозирования
Формула:

CPP
area = π × (diameter / 2)²
time = base_dwell_time × (area / area_0.5mm)
Где:

base_dwell_time — время для эталонной точки Ø0.5 мм
Результат ограничивается: 50 ≤ time ≤ 2000 мс
📌 Чем больше пад — тем дольше дозирование.

✅ Преимущества алгоритма
Функция	Выгода

Адаптивный диаметр	Корректная подача пасты для круглых и прямоугольных падов

Настройка параметров	Z, feed rate, offset — всё в одном классе
